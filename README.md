# 🧫 مساعد بحثي لدراسة اعتلال الكلى السكري عبر اللعاب
### مشروع بحثي وتعليمي — ISEF 2027

تطبيق ويب محلي (Local Desktop App) مبني بـ Streamlit، يعمل بالكامل على جهازك
(CPU/GPU) دون الحاجة لإرسال بياناتك لخوادم خارجية، مع دعم اختياري للبحث في
الويب عند توفر الإنترنت.

> ## ⚠️ تنويه أساسي وإلزامي حول الغرض من هذا البرنامج
> **هذا البرنامج مُصمَّم حصراً لأغراض الاستفادة والتطوير في البحث العلمي
> والتعلّم**، ضمن مشروع مدرسي/علمي لمسابقة ISEF. **هو ليس جهازاً طبياً
> معتمداً، وليس أداة تشخيص سريري، ولا يجوز استخدامه بأي شكل لتشخيص أو
> تقييم أو اتخاذ أي قرار علاجي بخصوص مريض حقيقي.** أي نتيجة يعرضها
> البرنامج — سواء كانت تحليلاً نصياً من نموذج ذكاء اصطناعي، أو نتيجة
> إحصائية، أو تنبؤ نموذج تعلّم آلي — هي **لأغراض تعليمية وبحثية فقط**،

> يجب دوماً مراجعة مختص طبي بشري مؤهل. هذا التنويه مُضمَّن أيضاً بشكل
> بارز داخل واجهة التطبيق نفسه (أعلى كل صفحة، وفي الشريط الجانبي، وفي كل
> تقرير PDF يُصدَّر).

## 🔧 خطوات التثبيت

### 1) تثبيت بايثون
تأكد من وجود Python 3.10 أو أحدث.

### 2) إنشاء بيئة افتراضية (موصى به)
```bash
python -m venv venv
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate
```

### 3) تثبيت المكتبات
```bash
pip install -r requirements.txt
```

### 4) تثبيت Ollama (للنمط الأوفلاين المحلي)
1. حمّل Ollama من: https://ollama.com/download
2. بعد التثبيت، افتح الطرفية ونفّذ:
   ```bash
   ollama pull llama3.2-vision
   ```
   (هذا يُنزّل النموذج مرة واحدة فقط، ثم يعمل محلياً بدون إنترنت لاحقاً)
3. تأكد أن Ollama يعمل في الخلفية (يبدأ تلقائياً عادة بعد التثبيت).

> 💡 إن كان جهازك ضعيفاً، يمكنك استخدام نموذج أخف مثل `llama3.2` (بدون رؤية)
> بدلاً من `llama3.2-vision`، لكن حينها لن يقرأ الصور مباشرة.

### 5) تشغيل التطبيق
```bash
streamlit run app.py
```
سيفتح تلقائياً في المتصفح على `http://localhost:8501`

## ☁️ النشر على Streamlit Cloud (بدون Ollama) — التحوّل التلقائي لنموذج مجاني

خادم Ollama **لا يعمل على Streamlit Cloud** لأنه يحتاج تشغيل عملية خلفية على
جهاز حقيقي، وهذا غير متاح في بيئة الاستضافة السحابية المجانية. لذلك أضفنا
منطق **تحوّل تلقائي وسلس (Automatic Fallback)** في `local_llm.py`:

1. التطبيق يحاول الاتصال بـ Ollama المحلي أولاً (بفحص سريع لا يتجاوز 2.5 ثانية).
2. إن فشل (كما هو الحال دوماً على Streamlit Cloud)، **يتحول تلقائياً وبصمت**
   لنموذج سحابي مجاني عبر **Groq API** (بدون أي رسالة خطأ توقف التطبيق).
3. إن لم يتوفر أي من الاثنين، يعرض التطبيق رسالة إرشادية هادئة في المحادثة
   نفسها (وليس صندوق خطأ Streamlit الأصفر) توضح للمستخدم كيف يفعّل أحد الخيارين.

### خطوات تفعيل البديل السحابي المجاني (Groq):

1. سجّل حساباً مجانياً بدون بطاقة ائتمان على: https://console.groq.com
2. من لوحة التحكم، أنشئ مفتاح API جديد (يبدأ بـ `gsk_...`).
3. في مشروعك على Streamlit Cloud: افتح **Settings → Secrets** وأضف السطر التالي:
   ```toml
   GROQ_API_KEY = "gsk_ضع_مفتاحك_هنا"
   ```
4. أعد تشغيل التطبيق (Reboot). سترى في الشريط الجانبي أن "المصدر النشط"
   تحوّل تلقائياً إلى "☁️ نموذج سحابي مجاني عبر Groq".

> 💡 نفس الخطوة تنطبق محلياً على جهازك أيضاً: أنشئ ملف `.streamlit/secrets.toml`
> في مجلد المشروع وضع فيه نفس السطر، أو عرّف متغير بيئة `GROQ_API_KEY` قبل التشغيل.

الفرق الوحيد عند استخدام Groq بدل Ollama: الاستدلال يحدث على خوادم Groq
السحابية (وليس على جهازك)، وسترى ملاحظة صغيرة في كل رد توضح أنه صدر عبر
النموذج السحابي الاحتياطي. جودة الردود قريبة جداً من Llama3.2 المحلي، مع
حد استخدام يومي مجاني سخي (راجع حدود الاستخدام الحالية على موقع Groq).

| الخاصية | كيف تعمل |
|---|---|
| الذاكرة الدائمة | كل عينة/محادثة تُحفظ في `data/ckd_saliva.db` (SQLite) ونسخة دلالية في `data/chroma_store/` (ChromaDB) لمقارنة العينات لاحقاً |
| نمط أوفلاين/أونلاين + تحوّل تلقائي | يحاول Ollama المحلي أولاً، وعند فشله (مثل النشر على Streamlit Cloud) يتحول تلقائياً وبصمت لنموذج Groq السحابي المجاني دون أي خطأ يوقف التطبيق |
| المعالجة متعددة الأنماط | تبويب "رفع العينات" يقبل صور + ملفات Excel/CSV + إدخال يدوي معاً |
| المقارنة الإحصائية | تبويب "المقارنة" يطبق ANOVA (لثلاث مجموعات) أو t-test (لمجموعتين) تلقائياً حسب البيانات المتوفرة |
| المجسم 3D | تبويب "المجسم ثلاثي الأبعاد" — تخطيطي توضيحي وليس تصويراً طبياً حقيقياً |
| الخبير الذكي | System Prompt مخصص في `local_llm.py` يجعل النموذج يتصرف كخبير استشاري، مع تنويهات دائمة بعدم اليقين وضرورة المراجعة البشرية |
| التحقق من الحقائق | خيار "Fact-Check" في المحادثة يبحث في الويب عن أدلة داعمة/مضادة لإجابة النموذج |
| محادثات متعددة | الشريط الجانبي يتيح فتح/تبديل عدة محادثات منفصلة، كل منها محفوظة في SQLite |
| إدارة الذاكرة العشوائية | تفريغ دوري تلقائي كل 5 تفاعلات عبر `gc.collect()`، مع زر يدوي وعرض استهلاك RAM عبر `psutil` |
| نموذج تعلّم آلي حقيقي | تبويب "نموذج التعلم الآلي" يدرّب `RandomForestClassifier` فعلياً على عيناتك المخزّنة (وليس فقط رأي LLM عام)، مع تقرير دقة، Cross-Validation، وأهمية المتغيرات — يحتاج 6 عينات على الأقل |
| تصدير تقرير PDF | زر "توليد تقرير PDF" في تبويب رفع العينات، يُصدِّر ملف PDF لكل عينة يتضمن القياسات وتحليل الذكاء الاصطناعي وتنبؤ نموذج التعلّم الآلي، مع تنويه الغرض البحثي مطبوع في أعلى وأسفل كل تقرير |
| تسجيل صوتي للملاحظات | حقل "سجّل ملاحظة صوتية" عند رفع عينة جديدة (`st.audio_input`)، يُحفظ الملف الصوتي محلياً ويُربط بملاحظة العينة |

## ⚠️ تنويه علمي وأخلاقي مهم

هذا التطبيق **أداة بحثية مساعدة** لمشروع مدرسي/علمي (ISEF)، وليس جهازاً
طبياً معتمداً أو بديلاً عن تشخيص طبي حقيقي. يجب توضيح هذه النقطة بوضوح
عند عرض المشروع في المسابقة، وعدم استخدام أي نتيجة يقدمها التطبيق لاتخاذ
قرارات تشخيصية أو علاجية فعلية على مرضى حقيقيين. النموذج قد يخطئ
(Hallucination)، لذلك أضفنا خاصية التحقق التلقائي وتنويهات "درجة الثقة
والقيود" في كل إجابة — لكنها لا تُلغي الحاجة لمراجعة مختص بشري مؤهل.

## ✅ خصائص أُضيفت مؤخراً (كانت أفكار تطوير، أصبحت مُنفَّذة)

- ✔️ **نموذج تعلّم آلي حقيقي** (`ml_classifier.py`): يدرّب `RandomForestClassifier` على
  العينات الفعلية المخزّنة، ويقيس دقته الحقيقية عبر Cross-Validation وتقرير تصنيف كامل،
  بدل الاعتماد فقط على رأي عام من النموذج اللغوي.
- ✔️ **تصدير تقرير PDF تلقائي** (`report_generator.py` عبر `fpdf2`): تقرير لكل عينة
  يتضمن القياسات، تحليل الذكاء الاصطناعي، وتنبؤ نموذج التعلّم الآلي، مع تنويه الغرض
  البحثي مطبوع في كل صفحة.
- ✔️ **تسجيل صوتي للملاحظات** أثناء رفع العينة عبر `st.audio_input`.
